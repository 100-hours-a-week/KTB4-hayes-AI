# HTTP
1. HTTP란
    클라이언트와 서버가 요청을 주고 받을 수 있게 미리 정의한 요청형식 약속
    
2. HTTP Message
    클라이언트와 서버의 요청/응답 메시지, 헤더와 바디로 이루어져 있다.
    
3. HTTP Request Method
    필요한 작업을 하기위한 요청 메소드인데 대표적으로 GET, POST, PUT, PATCH, DELETE가 있다.
    
4. HTTP Status Code
    요청 결과는 코드형식으로 나타낸다. 대표적으로 200 성공, 400 클라이언트 오류, 500 서버 오류가 있다.
    
5. HTTP URL
    URL을 입력하여 원하는 리소스를 요청하는 것이다.
    

# REST API
RSET 아키텍처 스타일을 따르는 API로, 일관된 규칙으로 쉽게 요청을 보내고 받을 수 있게 하는 형식이다.

# JSON
key와 value 쌍으로 이루어져 있는 데이터 형식인데, 다양한 환경에서 사용할 수 있도록 쓰는 표준 데이터 형식이다.
